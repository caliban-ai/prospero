# K8sFleet backend — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use `- [ ]`.

**Goal:** A `K8sFleet` backend of `FleetProvider` that drives a fleet via `CalibanTask` CRs and streams live sessions over the network, on top of the transport landed in prospero #71/#75.

**Architecture:** See [ADR 0008](../../adr/0008-k8s-fleet-backend.md). Builds on `#75`'s `CalibandClient::connect_tcp` + `Endpoint` + `AgentHandle.endpoint`. No transport work remains.

**Tech Stack:** Rust (edition 2024), tokio, `kube` + `k8s-openapi` (new, behind a `k8s` cargo feature), the existing `caliband::transport` + `Emitter`/bus/store.

## Global Constraints
- **Mirror the `CalibanTask` CRD, don't depend on the caliban-operator crate** (ADR 0003/0008 §1). Minimal fields only; golden-pin against a sample CR.
- `kube`/`k8s-openapi` are gated behind a `k8s` cargo feature; `LocalFleet` builds with no cluster.
- Session output must land in the **same event bus + `Store`** the API SSE/history read (ADR 0004) — no API-layer changes needed.
- Reuse the conformance suite (ADR 0007) via a `FakeBackend` trait; `LocalFleet`'s run stays behavior-identical.
- Match `k8s-openapi` to a stable feature aligned with the operator (e.g. `v1_32`).

## Tasks

### Task B1: `kube` deps + mirrored `CalibanTask` type
Add `kube` (client/runtime/derive) + `k8s-openapi` (`v1_32`) + `schemars` to `crates/core` behind `[features] k8s = [...]`. Create `crates/core/src/k8s/mod.rs` + `crates/core/src/k8s/crd.rs`: a minimal `#[derive(CustomResource)]` `CalibanTaskSpec` (`caliban.caliban-ai.dev/v1alpha1`, namespaced) with `workspace.sources: Vec<Source{name,repo,ref,path}>`, `task.prompt`, optional `isolation{runtimeClass}`; and `CalibanTaskStatus { phase: String, calibandEndpoint: Option<String>, sandboxRef: Option<NamedRef> }`. Golden test: a sample CR YAML (mirroring the operator's) round-trips. Gate the module on `feature = "k8s"`.

### Task B2: `K8sFleet` skeleton + `ensure_agent`/`stop_agent`/`restart_agent`
`crates/core/src/k8s/fleet.rs`: `struct K8sFleet { api: Api<CalibanTask>, namespace, emitter, settings }`. Implement `FleetProvider`:
- `ensure_agent`: name = `calibantask-<hash(spec)>`; SSA-apply via `Api::patch(Apply)`; poll (bounded) until `status.phase=="Running"` && `calibandEndpoint.is_some()`; return `AgentHandle { id, repo, endpoint: Endpoint::Tcp(calibandEndpoint) }`. Map errors → `CoreError`.
- `stop_agent(Kill)`: `api.delete(name)`. `Graceful`: delete + await gone within `timeout_ms`.
- `restart_agent`: delete + re-apply → fresh name/id.
Unit-test the pure spec→CR mapping + name determinism with a fake `Api` (or a trait-injected CR store) — no real cluster.

### Task B3: `K8sFleet::watch_fleet`
A `kube::runtime::watcher(api, watcher::Config::default())` → for each `Applied`/`Deleted`, diff `status.phase` against a local map and emit `FleetChange::{Discovered,StatusChanged,Gone}` (map operator `Phase`→`AgentStatus`: Provisioning→Spawning, Running→Running, Completed→Done, Failed→Failed). Seed the stream with an initial `list`. Return `BoxStream<'static, FleetChange>`.

### Task B4: Session-plane bridge (network attach → bus/store)
Extract the attach-loop core (`attach_once`/normalize/emit) from `FleetManager` into a provider-agnostic helper `caliband::attach::run_attach(client, endpoint, id, emitter)` if cheaper than duplicating; `K8sFleet` spawns it per Running agent using `CalibandClient::connect_tcp(host_port, tls_from_operator_ca, token)` and the shared `stream` normalizer + `Emitter`. Output lands in the same bus + `Store`. Integration test: stand up a TCP+TLS `FakeCaliband` (reuse `transport`'s test-gated `Listener`/`tls_server_from_pem`, `feature=testkit`) and assert frames reach the store under the agent's `stream_key`.

### Task B5: Generalize conformance + `FakeK8s`
In `testkit.rs`: `pub trait FakeBackend { fn received_any_spec(&self)->bool; fn simulate_reap(&self, id:&str); }`; change `fleet_provider_conformance`'s 2nd param to `&dyn FakeBackend`; impl for `FakeCaliband` (wrap `received_specs`/`remove_agent`); update the `LocalFleet` caller. Build `FakeK8s` — an in-memory `CalibanTask` store behind the same `Api` seam `K8sFleet` uses (trait-inject the CR client so tests need no apiserver), implementing `FakeBackend`. Add `k8s_fleet_satisfies_conformance` mirroring `setup_with_background_poll` (spawn `K8sFleet`'s watch loop against `FakeK8s`, run the suite).

### Task B6: Daemon backend selection + docs
`crates/daemon/src/main.rs`: select `LocalFleet` vs `K8sFleet` by `PROSPERO_FLEET=local|k8s` (+ `PROSPERO_K8S_NAMESPACE`, kubeconfig via `kube::Client::try_default`). Document in `docs/container.md`. Gate the k8s arm on `feature="k8s"`.

## Notes
- **Deferred (ADR 0008):** gRPC (#314); rerouting API-layer direct `FleetManager` calls; warm pools. Don't build.
- The `CalibanTask` field names/casing must match the operator's CRD (`caliban.caliban-ai.dev/v1alpha1`) — cross-check against `caliban-operator/deploy/crd/calibantask.yaml`.
- `k8s-openapi` version must be single across the build; if a conflict arises with a transitive dep, pin at the workspace.
